Difference between Library and Framework

Created On : 29 Aug 2021

Major differnece between Library and Framework is “Inversion Control”. When you call a library control is yours. But with framework it is inverted, the framework calls you.

  • Library

    • A library is just a collection of class definitions.

    • It is designed for code reusability. It will get the code that has already been written by other developers

    • It is already developed. People who need this just call this library and get the result without any code implementation

  • Framework

    • Framework is just basic structure of a system. We need to implement specific functionality based on our requirement.

    • In other terms, All the control flow is already there. There is a bunch of predefined white spots that you should fill out with our code

    • Developers no need to worry about whether design is good or not but just about implementation of domain specific functions.

  • Below is the example for a FrameWork

    Main.java - Entry point to the framework. It cannot be changed


    Move.java – It is a hook. A hook is an interface in a framework, where developers can defined/extend functions based on their own requirements.
